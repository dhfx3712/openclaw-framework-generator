# TOOLS.md - 框架生成器 指令工具清单

## 核心创建指令

### 1. /创建项目 【项目名】 【业务描述】
功能：一键生成完整 OpenClaw 项目框架（自动选择架构模式）
示例：
```
/创建项目 task-manager 团队任务管理与进度追踪系统
```
入参说明：
- 项目名：英文/小写，作为根目录名称
- 业务描述：项目用途、核心能力（自动匹配架构模式）

### 2. /创建项目 【项目名】 【业务描述】 【自定义规则】
功能：带自定义规则创建项目
示例：
```
/创建项目 data-pipeline 数据管道监控系统 数据底座:本地JSON文件,删除模式:硬删除
```
入参说明：
- 自定义规则：key:value 格式，多个用逗号分隔
- 支持规则：`数据底座`、`删除模式`、`节点标记`、`架构模式`

### 3. /创建项目 【项目名】 【业务描述】 架构模式:pipeline
功能：指定架构模式创建项目
示例：
```
/创建项目 log-analyzer 日志分析与异常检测系统 架构模式:pipeline
```

## 辅助查询指令

### 4. /查看模板规范
功能：展示标准目录结构、配置规则、模板文件说明

### 5. /查看已创建项目
功能：列出当前环境下所有已生成的 OpenClaw 项目目录

### 6. /查看模板
功能：查看 `templates/` 目录下的所有模板文件列表

### 7. /查看技能模板
功能：查看内置 Skill 模板列表（feishu-bitable-skill、notification-channel-skill、memory-system-skill）

### 8. /引入技能 【项目名】 【技能名】
功能：将内置 Skill 模板复制到已有项目中
示例：
```
/引入技能 my-project feishu-bitable-skill
/引入技能 my-project notification-channel-skill
```

### 9. /查看记忆框架
功能：查看三层记忆系统架构说明和使用规则

## 架构模式速查

| 模式 | 关键词触发 | 生成文件数 | 说明 |
|------|-----------|-----------|------|
| single-agent | 默认 | 11 | 1个Agent，直线型数据流 |
| hub-and-spoke | 管理/系统/调度/子Agent | 11 | 1主+N子，星型数据流 |
| pipeline | 管道/流水线/处理链 | 11 | N个顺序Agent，链型数据流 |
| daemon-agent | 查询/查词/翻译/引擎/高频/搜索 | 11 | 1 Agent + daemon，IPC 通信 |
| data-analysis | 分析/报表/SQL/数据库/指标/归因 | 11 | 1 Agent + SQL 模板 + 复核流程 |

## 补充说明
1. 生成规则：目录已存在则提示，不会覆盖原有文件
2. 初始状态：scripts 目录为空，业务脚本后续自行添加
3. 模板渲染：使用 `templates/` 目录下的通用模板，`{{变量名}}` 占位符自动替换
4. 架构约束：生成的项目自带 ARCHITECTURE.md（数据流+状态机+集成契约）
5. 配置校验：生成 config.schema.json，字段类型/必填/枚举均有约束
