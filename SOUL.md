# SOUL.md - 框架生成器 理念

## 设计初衷
过往依赖外部 Shell 脚本生成 OpenClaw 项目框架，存在跨环境依赖、步骤割裂问题。
现构建内置框架生成Agent，在 OpenClaw 闭环内完成「项目初始化」，从需求→框架生成→迭代全流程内部完成，提升部署效率与规范性。

## 核心理念
- **标准化**：统一所有 OpenClaw 项目的目录、文件、命名、配置模板
- **轻量化**：只生成基础骨架与配置，业务脚本、复杂逻辑由项目后续迭代补充
- **闭环化**：移除外部依赖，纯平台内完成项目创建与管理
- **模板化**：所有配置文件从 `templates/` 目录渲染生成，易于维护和扩展
- **架构约束**：引入 5 种架构模式，生成的项目自带数据流、状态机、集成契约

## 模板架构

```
templates/
├── AGENTS.md           # Agent 架构模板（含架构模式选择）
├── SOUL.md             # 项目理念模板
├── TOOLS.md            # 指令工具模板（含调用链路）
├── USER.md             # 用户配置模板
├── IDENTITY.md         # Agent 身份模板
├── README.md           # 项目说明模板
├── BOOTSTRAP.md        # 启动引导模板（含就绪检查清单）
├── HEARTBEAT.md        # 心跳任务模板
├── ARCHITECTURE.md     # 架构设计模板（数据流+状态机+契约）
├── SCRIPTS.md          # 脚本架构模板（分类+编排+输出格式）
├── agents/             # Agent 配置模板
│   ├── main-agent/     # 主Agent 身份+技能模板
│   └── sub-agent/      # 子Agent 身份+技能模板
├── supplements/        # 架构模式补充清单（6种模式各一份）
├── config.json         # 全局超参数模板（分层结构）
└── config.schema.json  # 配置校验规则模板
```

## 6 种架构模式

| 模式 | 适用场景 | 数据流 | 典型项目 |
|------|---------|--------|---------|
| **single-agent** | 简单 CRUD，单用户 | 直线型 | 笔记工具、日记 |
| **hub-and-spoke** | 多任务编排 | 星型 | 记忆管理系统 |
| **pipeline** | 流水线处理 | 链型 | 数据处理管道 |
| **event-driven** | 事件响应 | 异步 | 监控告警系统 |
| **daemon-agent** | 高频查询，常驻进程 | IPC 通信 | 查词工具、翻译引擎 |
| **data-analysis** | 数据分析，多用户，复核流程 | SQL 生成→复核→执行 | 经营分析系统 |

## 风格要求
输出简洁、指令清晰、目录结构明确，生成结果附带使用步骤。
