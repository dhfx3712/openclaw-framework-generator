# event-driven 模式 — 手动补充清单

## 架构说明
事件触发 → Agent 响应，异步数据流，适用于监控告警项目。

## 需要手动补充

### 🔴 高优先级（必须）
- [ ] **事件定义**：在 `TOOLS.md` 中定义所有触发事件（cron、webhook、消息等）
- [ ] **事件→Agent 映射**：在 `SOUL.md` 中定义每个事件由哪个 Agent 处理
- [ ] **业务指令定义**：在 `TOOLS.md` 中定义所有指令
- [ ] **数据底座配置**：在 `datas/config.json` 的 `storage.config` 中填写连接信息
- [ ] **SOUL.md 角色定位**：补充事件处理 Agent 的角色描述
- [ ] **IDENTITY.md 身份信息**：填写名称、定位、核心使命

### 🟡 中优先级（建议）
- [ ] **事件去重策略**：同一事件重复触发如何处理？
- [ ] **事件队列**：事件积压时如何处理？
- [ ] **输出格式规范**：在 `SCRIPTS.md` 中定义输出格式
- [ ] **状态机定义**：在 `ARCHITECTURE.md` 中定义实体状态流转
- [ ] **cron 任务注册**：通过 OpenClaw cron 模块注册定时触发事件

### 🟢 低优先级（按需）
- [ ] 业务脚本编写（`scripts/` 目录）
- [ ] 心跳任务配置（`HEARTBEAT.md`）
