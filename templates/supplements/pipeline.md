# pipeline 模式 — 手动补充清单

## 架构说明
N 个顺序 Agent，链型数据流，适用于流水线处理项目。

## 需要手动补充

### 🔴 高优先级（必须）
- [ ] **阶段定义**：在 `AGENTS.md` 中定义每个处理阶段的 Agent 职责
- [ ] **数据传递协议**：定义前一阶段的输出如何传递给后一阶段
- [ ] **业务指令定义**：在 `TOOLS.md` 中定义所有指令
- [ ] **数据底座配置**：在 `datas/config.json` 的 `storage.config` 中填写连接信息
- [ ] **SOUL.md 角色定位**：补充每个阶段 Agent 的角色描述
- [ ] **IDENTITY.md 身份信息**：填写名称、定位、核心使命

### 🟡 中优先级（建议）
- [ ] **错误中断策略**：某阶段失败时，是回滚还是跳过？
- [ ] **输出格式规范**：在 `SCRIPTS.md` 中定义各阶段输出格式
- [ ] **状态机定义**：在 `ARCHITECTURE.md` 中定义实体状态流转
- [ ] **cron 任务注册**：如有定时任务，通过 OpenClaw cron 模块注册

### 🟢 低优先级（按需）
- [ ] 业务脚本编写（`scripts/` 目录）
- [ ] 心跳任务配置（`HEARTBEAT.md`）
