# daemon-agent 模式 — 手动补充清单

## 架构说明
1 Agent + 守护进程，IPC 通信，适用于高频查询项目。

## 需要手动补充

### 🔴 高优先级（必须）
- [ ] **守护进程实现**：编写 `scripts/daemon/` 下的常驻进程代码
- [ ] **CLI 客户端实现**：编写 `scripts/cli/` 下的查询客户端
- [ ] **IPC 通信协议**：定义守护进程和客户端之间的通信格式（Unix socket / HTTP）
- [ ] **数据加载**：守护进程启动时加载的数据索引或缓存
- [ ] **SOUL.md 角色定位**：补充 Agent 的角色描述和行为准则
- [ ] **IDENTITY.md 身份信息**：填写名称、定位、核心使命

### 🟡 中优先级（建议）
- [ ] **守护进程管理**：启动/停止/状态查询命令
- [ ] **并发控制**：守护进程的最大并发连接数
- [ ] **输出格式规范**：在 `SCRIPTS.md` 中定义查询输出格式
- [ ] **编排规则**：在 `SCRIPTS.md` 中定义并发/串行编排逻辑
- [ ] **数据底座配置**：在 `datas/config.json` 的 `storage.config` 中填写

### 🟢 低优先级（按需）
- [ ] 数据构建脚本（`scripts/build/`）
- [ ] 安全加固脚本（`scripts/security/`）
- [ ] 心跳任务配置（`HEARTBEAT.md`）
