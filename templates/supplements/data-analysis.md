# data-analysis 模式 — 手动补充清单

## 架构说明
单 Agent + SQL 模板 + 语义层 + 多用户角色 + 复核流程，适用于企业级数据分析项目。

## 需要手动补充

### 🔴 高优先级（必须）
- [ ] **SQL 模板目录**：创建 `templates/` 目录，编写业务 SQL 模板文件
  - 每个模板一个 `.sql` 文件，使用 `${参数名}` 占位动态参数
  - 示例：`gmv_abnormal.sql`、`gmv_dimension.sql`
- [ ] **语义层配置**：创建 `semantic.yaml`，定义：
  - 指标映射（用户表述 → 数据库字段）
  - 维度映射（用户表述 → 数据库字段）
  - 维度枚举值校验
  - 时间筛选规则
  - 表关联配置
- [ ] **数据库配置**：创建 `.env` 文件，填写数据库连接信息
- [ ] **建表 SQL**：编写数据库建表语句和测试数据
- [ ] **多用户角色定义**：在 `AGENTS.md` 中定义各角色职责
  - 业务分析师：发起分析指令
  - 数据工程师：维护数据和模板
  - 复核员：审核 SQL
  - 管理员：管理权限和审批
- [ ] **用户权限和流程**：在 `USER.md` 中定义用户分类、权限、操作流程
- [ ] **复核流程**：在 `SOUL.md` 中定义 SQL 复核流程（提交→审核→通过/驳回→执行）
- [ ] **SOUL.md 角色定位**：补充 Agent 的角色描述和行为准则
- [ ] **IDENTITY.md 身份信息**：填写名称、定位、核心使命

### 🟡 中优先级（建议）
- [ ] **SQL 生成目录**：创建 `generated_sql/` 目录，归档生成的 SQL
- [ ] **输出格式规范**：定义分析结果的展示格式
- [ ] **cron 任务注册**：如有定时分析任务，通过 OpenClaw cron 模块注册
- [ ] **异常处理流程**：定义指令无法解析、SQL 执行失败等异常处理方式

### 🟢 低优先级（按需）
- [ ] 数据校验脚本（`scripts/` 目录）
- [ ] 报告生成模板
- [ ] 心跳任务配置（`HEARTBEAT.md`）
