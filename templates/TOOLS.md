## 指令 → Agent → 脚本 调用链路

```
用户指令
  └→ 主Agent 解析
       ├→ 直接处理（简单查询/状态变更）
       └→ 调度子Agent（复杂业务）
            └→ 调用业务脚本（exec）
                 └→ 读写存储
                      └→ 返回结果 → 逐级回传 → 用户
```

---

## 指令开发规范

新增指令时需遵循：
1. 指令以 `/` 开头
2. 主Agent 统一接收和路由
3. 涉及存储操作 → 通过子Agent/脚本处理
4. 涉及状态变更 → 遵守状态机定义
5. 返回格式统一：`[状态] 操作结果描述`
